1. O que é o Emacs?

  • Emacs é uma família de editores popularmente conhecida pela sua extensibilidade.
  • O GNU Emacs é a variação mais conhecida e adotada.
  • O Emacs começou a ser desenvolvido em meados dos anos 70 no labotório de interligência artificial do MIT.
  • Já o GNU Emacs, começou a ser desenvolvido em 1984 pelo Richard Stallman, como uma alternativa livre ao Gosling Emacs (do James Gosling, pai do Java) que era proprietário.
  • Outras variações populares: XEmacs e SXEmacs.

2. Domine suas ferramentas

We’ve talked before about tools being an extension of your hand. Well, this applies to editors more than to any other software tool. You need to be able to manipulate text as effortlessly as possible, because text is the basic raw material of programming.

—Pragmatic Programmer

3. Frames, Windows & Buffers

3.1. Frame

  • É uma representação visual do agrupamento de janelas (Windows).
  • Um Frame pode ter uma ou mais janelas.

3.2. Window

  • É onde um Buffer é trabalhado.
  • Comparado com outros editores, visualmente a Window é como se fosse um painel.

3.3. Buffer

  • É a representação do texto na memória do Emacs.
  • É o contexto onde um texto é manipulado.

4. Major mode & Minor mode

4.1. Major

  • É o como o Emacs se especializa com recursos para edição de texto (Buffer).
  • Cada Buffer pode ter apenas um Major mode ativo por vez.

4.2. Minor

  • São modos opcionais que alteram características específicas na edição de texto.
  • Os buffers podem ter zero ou mais Minor modes ativos por vez.
